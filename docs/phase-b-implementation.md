# Phase B: 논문 평가 및 요약 파이프라인 구현 상세 (paper-service)

본 문서는 `paper-service`에서 구현된 논문 자동 평가, 요약 및 하이브리드 데이터 저장 파이프라인에 대한 상세 내용을 정리합니다.

## 1. 아키텍처 개요

`paper-service`는 모니터링 시스템(A)이 감지한 신규 논문 이벤트를 Kafka를 통해 수신하여, AI 전문가 수준의 평가와 요약을 수행하고 그 결과를 RDB(MariaDB)와 VectorDB(ChromaDB)에 분산 저장합니다.

### 데이터 흐름
1.  **Kafka 수신**: `new_papers_topic` 토픽에서 논문 메타데이터 및 PDF URL 수신.
2.  **논문 평가 (Evaluation)**: AI 도메인 적합성 및 품질 평가 (앙상블 방식).
3.  **PDF 파싱**: 평가 통과 논문에 한해 원본 PDF 다운로드 및 전체 텍스트 추출.
4.  **AI 요약 (Summarization)**: 추출된 본문을 바탕으로 마크다운 형식의 고품질 요약본 생성.
5.  **사내 문서 매핑**: 요약본을 기준으로 `internal_docs`와 유사도 분석 및 연관 관계 도출.
6.  **하이브리드 저장**: MariaDB(프론트용) 및 ChromaDB(검색용)에 각각 최적화된 형태로 적재.

---

## 2. 주요 구현 모듈

### 2.1. AI Scientist 방식의 고도화된 자동 리뷰어 (`evaluator.py`)
-   **앙상블 평가 (Ensemble)**: 5명의 독립적인 가상 리뷰어(GPT-4o)가 각각 논문을 독립적으로 분석합니다.
-   **자기 반성 루프 (Reflection)**: 각 리뷰어는 자신의 초안을 스스로 비판하고 수정하는 과정을 5회 반복하여 리뷰의 논리적 엄밀성을 확보합니다.
-   **Area Chair 시스템**: 최종적으로 총 25개의 리뷰 단계(5인 * 5회 반성)를 종합하여 에어리어 체어가 최종 `Accept/Reject` 결정 및 합의된 점수를 산출합니다.
-   **비판적 페르소나**: "낙관적 편향"을 배제하기 위해 논문의 결함과 한계를 찾아내는 데 집중하는 엄격한 비판자 모드로 동작합니다.
-   **풀텍스트 기반 분석**: PDF에서 추출된 전체 본문(최대 15,000자)을 직접 읽고 평가하여 초록만으로 알 수 없는 기술적 세부 사항까지 검토합니다.

### 2.2. Markdown 기반 요약 시스템 (`summarizer.py`)
-   프론트엔드 대시보드에서 즉시 렌더링 가능하도록 **Markdown 문법**을 사용하는 요약본을 생성합니다.
-   `shared/schemas/paper_summary.schema.json` 규격을 100% 준수하는 JSON 구조로 응답을 반환합니다.

### 2.3. 사내 문서 유사도 분석 및 순위 누적 (`consumer.py`)
-   ChromaDB의 `internal_docs` 컬렉션을 검색하여 논문과 가장 연관성이 높은 사내 문서를 최대 10개까지 도출합니다.
-   **중복 병합 로직**: 동일한 문서 내의 여러 청크가 검색될 경우, 순위를 중복해서 매기지 않고 해당 문서의 `reason` 필드에 청크 내용을 마크다운 형태로 계속 누적(Append)합니다.

## 3. 하이브리드 데이터베이스 전략

본 프로젝트는 데이터의 용도에 따라 RDB와 VectorDB를 동시에 사용하는 하이브리드 구조를 채택했습니다.

### 3.1. MariaDB (RDB) - "프론트 및 관계 저장소"
-   **역할**: 프론트엔드 사용자에게 정보를 빠르고 예쁘게 보여주기 위한 용도.
-   **저장 데이터**:
    *   `paper_summary` 테이블: 논문 정보, **마크다운 요약 본문**, 저자, URL, 카테고리.
    *   `paper_relate` 테이블: 논문-사내 문서 간의 확정된 순위 및 매칭 사유.
-   **장점**: 트랜잭션 보장, 정형 데이터 조회 속도 최적화, 마크다운 텍스트 보존.

### 3.2. ChromaDB (`papers` 컬렉션) - "기계 학습 및 분석 엔진"
-   **역할**: 시스템 내부의 지능형 검색 및 분석을 위한 용도.
-   **저장 데이터**: 논문 요약본의 임베딩 벡터 및 메타데이터.
-   **구현 이유**:
    1.  **의미 기반 검색(Semantic Search)**: "이 논문과 내용상 유사한 다른 논문 추천" 기능 구현 가능.
    2.  **지능형 Q&A (RAG)**: 사용자가 수집된 논문들에 대해 질문할 때 관련 문장을 즉시 추출하는 근거 자료 저장소로 활용.
    3.  **확장성**: 추후 논문 간의 차이점 분석이나 트렌드 분석 시 벡터 연산 속도 확보.

## 4. 인터페이스 및 테스트

-   **프론트엔드 API**: `GET /papers` 엔드포인트를 통해 RDB/VDB에 통합 저장된 최신 요약 리스트를 반환합니다.
-   **통합 테스트**: `test_integration.py`를 통해 모니터링(A)부터 저장(B)까지의 전 과정을 자동 검증할 수 있습니다.
